# On-premise vs Cloud

## On-Premise

- Softwares estão instlados localmente na empresa ou em um datacenter
- Custo inicial alto
- Hardware possui depreciação, é necessário considerar problemas técnicos no orçamento
- Hardware exige manutenção
- Precisa de profissionais qualificados com conhecimentos em hardware, rede, software, virtualização, etc.
- Escalabilidade complexa
- Alta disponibilidade complexa

### Vantagens:

- Alto controle sobre o softwae, hardware, protocolos etc
- Altamente customizável
- Acesso físico a máquina
- Hardware normalmente é mais barato que cloud
- Controle dos dados
- Integração com sistema legados
- Compliance e regulações
- Custos previsíveis
- Sem lock-in, não depende de um cloud provider
- Se bem dimensionado, o custo inicial se paga ao longo dos anos e pode ser menor do que Cloud

## Cloud Computing

- Custo alto ao longo do tempo para determinados serviços
- Alto custo para transferência de dados (exemplo transferir dados de um S3 para uma CND, ou de cloud provider para o outro, para a aplicação etc)
- Previsibilidade de custos mais complexas
- Vendor lock-in (dificil de migrar de um para o outro)
- Riscos adicionais de segurança
- Controle limitado para fazer upgrade
- Compliance
- Integração pode ser mais complexa com sistema legados

### Vatagens:

- Baixo custo inicial
- Escalabilidade de forma simplificada
- Acessibilidade (infraestrutura, tecnologias avançadas etc que não seriam viáveis no on-premises)
- Alta disponibilidade
- Não há custo com profissionais especilizados para cuidar do hardware, rede, etc
- Pay as you go
- Backup automazidos
- Serviços gerenciados
- Recuperação rápida em casos de desastres
- Amplitude de serviços

## Modelos Híbridos

Pode ser interessante em uma transição ou quando precisa de soluções específicas para cada contexto, mas pode usufruir das vantagens do outro modelo também.

- Redução de custos para grande utilização sazonal
- Integração de serviços pode ser complexa
- Pode ter grande latência
- Deve ter profissionais especializados em diversas áreas, pois precisará de profissionais para cada modelo
- Outra possibilidade é utilizar diferentes provedores ao mesmo tempo
